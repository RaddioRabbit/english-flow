# 产品需求文档（PRD）

## 英语长难句解析一键生成 Agent 系统

---

## 产品概述

### 产品定位

英语长难句解析一键生成 Agent 系统是一款面向英语学习者、雅思备考者和英语教育内容创作者的智能化工具。该系统能够自动解析英语原著中的长难句，生成包含句译对照、句式分析、词汇解析和雅思备考建议的专业教学条漫图片集。

### 目标用户

1. **雅思备考学生**：需要系统学习长难句解析方法，提升听说读写能力
2. **英语专业学生**：需要深入理解英语语法结构和词汇用法
3. **英语教育内容创作者**：需要快速生成高质量的英语教学图文内容
4. **英语教师**：需要制作教学课件和辅助材料
5. **英语原著阅读爱好者**：希望深入理解经典文学作品中的复杂句式

### 核心价值

- **一键生成**：输入英语句子和原著信息，系统自动完成翻译、分析、图片生成全流程
- **专业解析**：基于大语言模型的深度语法分析，提供时态、语态、句式结构的专业解读
- **视觉呈现**：生成小红书风格的教学条漫，信息图表化，易于理解和传播
- **多维度学习**：覆盖句译对照、句式分析、词汇解析、雅思备考四大学习维度

### 解决的问题

1. 长难句理解困难，缺乏系统的解析方法
2. 手动制作教学图文耗时耗力
3. 传统英语学习资料枯燥，缺乏视觉吸引力
4. 雅思备考缺乏针对性的长难句训练材料

## 技术架构

### 技术栈

- **前端框架**：Next.js 14 (App Router)
- **后端服务**：Node.js + Supabase
- **Agent 框架**：Claude Agent SDK
- **大语言模型**：Kimi API (Anthropic 兼容 API)
- **图像生成**：Gemini 3.1 Flash Image (Nano Banana 2)
- **包管理器**：pnpm
- **部署平台**：Vercel

## 页面功能详述

### 1. 首页 / 输入页面

#### 页面功能

提供用户输入英语句子和原著信息的界面，启动一键生成流程。

#### UI布局

- **顶部导航栏**：Logo、导航链接（首页、历史记录、关于）
- **主输入区域**：
  - **[输入框1] 英语原句**：多行文本输入框，支持粘贴长难句
  - **[输入框2] 原著书名**：单行文本输入，如 "The Great Gatsby"
  - **[输入框3] 原著作者**：单行文本输入，如 "F. Scott Fitzgerald"
  - **[按钮] 开始生成**：主按钮，触发工作流
- **示例区域**：
  - 展示几个经典例句供用户快速体验
  - 如：《双城记》开篇句、《了不起的盖茨比》经典句等
- **底部说明**：
  - 生成流程说明（翻译 → 分析 → 生成5张图片）
  - 预计耗时提示（约2-3分钟）

#### 交互逻辑

- **[操作1] 输入英语句子**：
  - 系统实时检测句子长度，超过200字符提示"长难句"
  - 自动去除首尾空白字符
  
- **[操作2] 点击示例句子**：
  - 自动填充三个输入框
  - 示例句子预置经典文学名句

- **[操作3] 点击开始生成**：
  - 验证所有必填字段
  - 创建任务记录到 Supabase
  - 调用 Master Agent 启动工作流
  - 页面跳转到任务详情页

### 子 Agent 详细说明

#### Agent 1: 翻译 Agent (Translation Agent)

**功能**：生成英语句子的汉语翻译

**输入**：
- 英语原句
- 原著书名
- 原著作者

**输出**：
- 汉语翻译文本

**提示词模板**：
```
你是一位精通英语文学的翻译专家。请将以下出自《{book}》的英语句子翻译成地道、准确的中文：

原句：{sentence}

要求：
1. 忠实原文含义，保持文学性
2. 符合中文表达习惯
3. 保留原文的语气和风格

请直接输出翻译结果，不要添加解释。
```

---

#### Agent 2: 分句 Agent (Segmentation Agent)

**功能**：根据句子长度将原句和翻译划分到四宫格

**输入**：
- 英语原句
- 汉语翻译

**输出**：
- prompt1: 英文原句第一部分
- prompt2: 中文翻译第一部分
- prompt3: 英文原句第二部分（如需要）
- prompt4: 中文翻译第二部分（如需要）

**逻辑**：
- 句子长度 < 100字符：使用2宫格（prompt1, prompt2）
- 句子长度 100-200字符：使用4宫格（prompt1-4）
- 句子长度 > 200字符：智能拆分为主句+从句结构

---

#### Agent 3: 图片描述提示词 Agent (Image Prompt Agent)

**功能**：生成符合原文场景的图片生成提示词

**输入**：
- 英语原句
- 原著书名
- 上下文语境

**输出**：
- 英文图片描述提示词（用于 Nano Banana 生成）

**提示词模板**：
```
你是一名会prompt的配图大师，如果你要给《{book}》的这句原文，
结合这句原文的语境上下文：{origin}，配一张符合原文描述的图片，
且不可怕的场景，在nano banana生成这张图片，你会怎么写提示词
```

---

#### Agent 4: 语法精析 Agent (Grammar Analysis Agent)

**功能**：生成句子的详细语法分析

**输入**：
- 英语原句
- 原著书名

**输出**：
- 时态分析
- 语态识别
- 句式分析（含完整结构总结）

**提示词模板**（用户已提供）：
```
#### 语法精析 

**时态分析**
全句使用**[时态名称]**（[具体动词形式]），用于描述[该时态的功能和作用]

**语态识别**
**[主动语态/被动语态]**（[具体动词示例]）

**句式分析**

- [句子成分1]：[详细说明]
- [句子成分2]：[详细说明]
- [句子成分3]：[详细说明]
- **[完整句式结构总结，必须加粗]**
```

---

#### Agent 5: 词汇解析 Agent (Vocabulary Agent)

**功能**：解析句子中的6个核心/难点词汇

**输入**：
- 英语原句
- 原著书名
- 上下文

**输出**：
- 6个单词的详细解析，每个包含：
  - 词汇原形 [音标]
  - 词性
  - 释义
  - 例句
  - 译文

**提示词模板**（用户已提供）：
```
#### 词汇解析

[解析句中的核心词汇和难点词汇，每个词汇必须包含以下信息：
- 编号（如 1., 2., 3.）
- **词汇原形** [音标]
  - 词性：[名词/动词/形容词/副词等]
  - 释义：[中文释义]
  - 例句：[英文例句]
  - 译文：[例句的中文翻译]

注意：本句至少解析 {vocab_count} 个核心词汇，重点关注《{book_name}》中的特殊用法、高级词汇、固定搭配等]

参考上下文信息：{context}
```

**功能**：生成针对雅思听说读写四项的备考建议

```

### 图像生成 Agent 集群

#### Page 1-1 Agent: 句译对照图生成

**功能**：生成6宫格句译对照教学条漫

**输入**：
- book: 书名
- origin: 原文语境
- prompt1-4: 分句后的英汉对照

**输出**：
- 6宫格图片（3:4比例，2K分辨率）

**工作流程**：
1. 调用 LLM 生成图片描述提示词
2. 组合最终图像生成提示词
3. 调用 Gemini Image Generation Skill 生成图片

**参考代码**：`english_page1_1_agent.js`

#### 语法精析 

**时态分析**
全句使用**[时态名称]**（[具体动词形式]），用于描述[该时态的功能和作用]

**语态识别**
**[主动语态/被动语态]**（[具体动词示例]）

**句式分析**

- [句子成分1]：[详细说明]
- [句子成分2]：[详细说明]
- [句子成分3]：[详细说明]
- **[完整句式结构总结，必须加粗]**

【格式说明及示例】：

**时态分析格式示例**：
全句使用**一般过去时**（was, lay），用于描述过去发生的状态和存在。
或：全句使用**现在完成时**（have done），用于强调过去动作对现在的影响。

**语态识别格式示例**：
**主动语态**（the road lay / it lumbered）
或：**被动语态**（was written / were made）

**句式分析格式示例**：
- 主句：The Dover road lay beyond the Dover mail
- 插入语：as to him（介词短语作插入语，表示"就他而言"）
- 方式状语从句：as it lumbered up Shooter's Hill（由as引导，表示"当...时候"）
- **主语（The Dover road） + 谓语（lay） + 表语（beyond the Dover mail） + 插入语（as to him） + 方式状语从句（as it lumbered...）**

【关键要求】：
1. 时态名称、语态名称必须用**加粗**
2. 句式分析最后一行的完整结构总结必须用**加粗**
3. 格式必须与上述示例完全一致
```

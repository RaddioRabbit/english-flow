# 产品需求文档（PRD）

## 一、产品概述

### 1.1 产品名称

英语原著句子解析图片 Agent 一键生成平台

### 1.2 产品定位

这是一个面向英语教育内容创作者、英语原著解析博主、雅思教学博主、英语课程团队的 Agent 化内容生产平台。用户输入一条英语句子、该句来源原著、原著作者，并可选上传参考图片，系统即可自动完成文本解析与系列图片生成，输出适合小红书、抖音、公众号、课程讲义等场景使用的图文内容。

### 1.3 目标用户

核心用户包括英语教学博主、英语原著精读内容创作者、雅思老师、英语培训机构内容运营人员，以及需要批量生成教学类图文素材的团队。

### 1.4 核心价值

本产品将“句子理解—中文翻译—句式分析—词汇解析—雅思备考—图片提示词生成—图片生成”这一整条复杂内容生产链标准化、模板化、Agent 化，帮助用户从零散的人工提示词编写和多工具拼接，升级为一键触发、统一风格、可复用的内容生产系统。

### 1.5 解决的问题

传统英语句子解析内容生产高度依赖人工：人工拆句、人工翻译、人工写解析、人工整理排版、人工调用图片工具，流程长、耗时高、风格不一致、难以批量复用。本产品通过 Claude Agent SDK 风格的工作流编排、Kimi Code 大模型解析、独立图像生成 skill、Supabase 存储与任务管理，将整条链路产品化。

### 1.6 技术栈约束

前端基于 Next.js，项目初始化方式为 `pnpm create next-app --example with-supabase with-supabase-app`。
鉴权、数据库、对象存储使用 Supabase。
Agent 编排使用 Claude Agent SDK 思路，在 Node.js 环境中实现。
文本解析模型使用 Kimi Code，并封装为可替换的模型适配层。
图像生成通过独立的生成图片 skill 调用。
系统支持参考图引导生成图片。

------

## 二、产品核心业务流程

用户输入英语句子、原著名、作者名。系统首先调用 Kimi Code 生成中文翻译。随后根据句子长度自动规划第一张“句译对照”图片中的四宫格内容分布（宫格一：原句部分一；宫格二：原句部分一对应的中文翻译；宫格三：原句部分二；宫格四：原句部分二对应的中文翻译），并结合句意生成适合生图模型的场景提示词。若用户上传了参考图片，则系统先对参考图片进行视觉特征分析，再把这些结果融合进图片提示词中，得到更稳定、更贴近用户预期的生成结果。

在此基础上，系统继续生成句式分析内容，严格遵循“时态分析—语态识别—句式分析—完整结构总结”的固定格式；然后拆分生成“句式分析图”和“句式总结图”。接着系统抽取六个重点生疏词，生成词汇解析文本，并制作“词汇解析图”。然后再生成“雅思备考”文本，最后产出“雅思备考图”。全部文本和图片结果会统一写入数据库并在结果页展示。

## 三、页面功能详述

### 3.1 首页 / Agent 创建页

#### 页面功能

这是系统主入口。用户在本页创建一次新的英语句子解析任务，配置要生成的模块、图片模式、参考图等，并一键启动 Agent 工作流。

#### UI布局

- **核心输入区**：输入英语句子、原著名称、作者名称，以及一个“一键解析”按键，Kimi Code 对原英语句子解析，生成对句子的“原句翻译”、“句式分析”、“词汇解析”、“雅思备考”的解析。
- **句子解析区**：将“核心输入区”点击的“一键解析”按键，生成的对句子的“原句翻译”、“句式分析”、“词汇解析”、“雅思备考”的解析情况进行显示，用户如果对解析结果不满意，可以再到“核心输入区”，点击“一键解析”，对原英语句子进行再解析，或者手动进行修改。
- **参考图上传区**：支持上传 1 到 5 张参考图片，显示缩略图、文件状态、删除按钮，每张参考图片的传送的下方要写有“句译对照图参考”、“句式分析图参考”、“句式总结图参考”、“词汇解析图参考”、“雅思备考图参考”，用来对应生成“句译对照图”、“句式分析图”、“句式总结图”、“词汇解析图”、“雅思备考图”。注意：如果用户没有删除这张图片，默认用户上传了图片之后，是保存了这张图片，“记住”了这张图片，刷新，或者重新登录之后是可以还是显示原来上传到图片的。
- **模块勾选区**：支持勾选是否生成句译对照图、句式分析图、句式总结图、词汇解析图、雅思备考图。
- **提交操作区**：包含“一键生成 Agent”按钮，用来一键按照参考图生成五张图片。
- **图片成果区**：分“句译对照图”、“句式分析图”、“句式总结图”、“词汇解析图”、“雅思备考图”五个展示图片的区域，每个区域都有一个任务进度条，可以分别显示图片任务生成的进度情况，以及展示五类已经生成图片的缩略图，以及图片的下载，如果图片生成有错误，展示失败节点、错误摘要。

#### 交互逻辑

- 用户输入必填内容后，系统进行前端校验，缺失则禁止提交。
- 用户上传参考图后，系统立即执行格式、大小、尺寸校验。
- 用户一类参考图片对应一类图片，“句译对照图参考”只能对应“句译对照图”，“句式分析图参考”只能对应“句式分析图”，“句式总结图参考”只能对应“句式总结图”，“词汇解析图参考”只能对应“词汇解析图”，“雅思备考图参考”只能对应“雅思备考图”。
- 用户点击“一键生成 Agent”后，系统创建任务记录并进入执行流程。
- 用户点击“保存为模板”后，系统将当前配置保存为个人模板。

------

### 3.2 登录与账户页

#### 页面功能

负责用户登录、额度显示、模型状态展示、个人信息管理。

#### UI布局

- **登录区**：支持 Supabase Auth 登录。
- **账户信息区**：展示昵称、邮箱、注册时间。
- **额度区**：展示剩余生成次数、图片额度、任务数量。
- **模型与技能状态区**：展示 Kimi Code 状态、图片 skill 状态、默认模板版本。

#### 交互逻辑

- 未登录用户不能创建任务。
- 登录后系统自动加载历史任务和个人模板。
- 当额度不足时，系统阻止提交并提示原因。
- 若模型或图片 skill 配置异常，页面给出状态提醒。

## 四、全局交互说明

### 4.1 导航结构

系统采用顶部导航结构。首页负责创建任务，任务详情页负责跟踪执行，结果页负责消费成果，模板页负责管理提示词，历史页负责复用任务，参考图页负责管理风格资产，账户页负责个人设置和配额。

### 4.2 登录流程

系统基于 Supabase Auth，支持邮箱或 OAuth 登录。用户未登录时仅可查看产品说明，登录后方可提交任务、保存模板、查看历史记录和下载结果。

### 4.3 Agent 执行流程

创建任务后，系统写入 `agent_tasks` 记录，状态初始为 `queued`。后端 Orchestrator 拉取任务并逐步执行。每个执行节点均记录输入、输出、状态、耗时、错误信息，支持断点重试和部分重跑。

### 4.4 图片生成模式

参考图引导生图模式，系统先对参考图做视觉分析，再将分析结果融合进 prompt，以获得更高一致性的图片结果。

### 4.5 业务规则

英语句子、原著名、作者名为必填。
默认生成完整五类图片，但用户可关闭部分模块。
词汇解析默认输出 6 个生疏词。
雅思备考固定按听力、口语、阅读、写作四段输出。
图片默认采用统一画布风格，适配内容平台发布。

### 4.6 参考图规则

用户可上传 1 到 5 张参考图。
支持 JPG、PNG、WEBP。
可设置参考风格、参考构图、参考角色服饰、参考排版布局等策略。

### 4.7 异常处理

若文本生成失败，任务进入文本失败状态并保留中间日志。
若文本成功而图片失败，则进入部分成功状态，可只重试图片。
若参考图无效，则提示原因并降级为普通模式。
所有失败信息都需面向用户可读。

## 五、内容生成规格

### 5.1 中文翻译模块

输入为英语句子、原著名、作者名。
输出为准确、自然、适合教学场景的中文翻译。
该翻译将用于第一张句译对照图。

### 5.2 四宫格分配模块

系统根据句子长度、停顿位置和中文对应关系，自动将原句与译文切分到图片四宫格中。
目标是保证每格文本信息量均衡、阅读顺畅、视觉美观。

### 5.3 场景提示词模块

系统基于句子内容生成适合图片模型理解的场景提示词。
若存在参考图，则融合参考图的视觉特征，生成最终 prompt。

### 5.4 句式分析模块

必须严格遵循以下格式：

#### 语法精析

**时态分析**
全句使用**[时态名称]**（[具体动词形式]），用于描述[该时态的功能和作用]

**语态识别**
**[主动语态/被动语态]**（[具体动词示例]）

**句式分析**

- [句子成分1]：[详细说明]
- [句子成分2]：[详细说明]
- [句子成分3]：[详细说明]
- **[完整句式结构总结，必须加粗]**

系统必须保证时态名称、语态名称、完整结构总结为加粗形式，格式稳定，可直接用于图片渲染。

### 5.5 词汇解析模块

系统至少解析 6 个重点词汇。每个词汇必须包含：
编号、词汇原形、音标、词性、中文释义、英文例句、中文译文。
重点优先选择原著中的生疏词、高级词汇、特殊用法、固定搭配。

### 5.6 雅思备考模块

输出必须以自然段形式呈现，可分为四段，对应听力、口语、阅读、写作。
禁止使用列表。
每段都要给出真实可用的备考分析，而不是空泛评价。

## 六、图片产出规格

### 6.1 句译对照图

内容包括英文原句切分、中文翻译切分、句子氛围图像。
支持四宫格布局。
图片生成前需要先完成翻译、切分和场景 prompt。

### 6.2 句式分析图

内容包括时态分析、语态识别、句式分析要点。
重点突出结构拆解和主干逻辑。

### 6.3 句式总结图

内容为完整句式结构总结。
强调一句话看懂全句框架，适合做单独总结页。

### 6.4 词汇解析图

内容包括 6 个重点词汇解析。
布局需适配信息密度，便于用户逐词阅读。

### 6.5 雅思备考图

内容包括听力、口语、阅读、写作四部分。
图片应兼顾信息完整度和平台阅读体验。

### 6.6 参考图引导生成

若用户上传参考图，则图片生成模块优先参考用户指定的风格、构图、色彩、服饰、布局等视觉特征。
系统应支持“仅参考风格”与“参考风格+构图”等不同模式。

## 七、系统模块设计

### 7.1 前端模块

基于 Next.js App Router 构建。负责表单输入、任务提交、状态轮询、结果展示、模板编辑、参考图库管理。

### 7.2 Agent Orchestrator

后端核心编排层，统一调度所有节点。
负责读取任务配置、组织执行顺序、管理状态流转、处理重试和降级。

### 7.3 Prompt Registry

统一管理所有提示词模板，支持版本化、测试、发布和回滚。

### 7.4 Model Adapter

封装 Kimi Code 接口，统一暴露文本生成能力。
业务层只调用统一接口，不直接依赖具体模型供应商实现。

### 7.5 Reference Image Analyzer

负责解析参考图的风格、构图、色彩、人物关系、服装特征、版式特征，并输出结构化摘要。

### 7.6 Reference-aware Prompt Composer

负责将句意 prompt 与参考图分析结果合成为最终图片 prompt。

### 7.7 Skill Adapter

负责调用生成图片 skill。
需要兼容纯文本生图和参考图引导两类调用方式。
若 skill 暂不支持直接图像条件输入，则采用“参考图分析结果转文字约束 prompt”的兼容方案。

### 7.8 Persistence Layer

使用 Supabase 存储用户、任务、步骤、模板、资源和配额日志。

------

## 八、数据库设计建议

### 8.1 profiles

存用户资料、角色、额度、默认模板版本。

### 8.2 agent_tasks

存任务主记录，包括输入内容、任务状态、模块配置、图片模式、模板版本、时间戳。

### 8.3 agent_task_steps

存任务中的每个步骤，包括步骤名、输入、输出、状态、错误信息、耗时、重试次数。

### 8.4 prompt_templates

存模板名称、类型、正文、变量说明、版本号、启用状态。

### 8.5 generated_assets

存生成后的文本和图片资源，包括资源类型、URL、任务归属、参考图关联信息。

### 8.6 reference_images

存用户上传的参考图，包括文件信息、标签、尺寸、启用状态、使用记录。

### 8.7 reference_image_analysis

存每张参考图的结构化分析结果，包括风格关键词、构图关键词、配色关键词、角色关键词、布局关键词和摘要。

### 8.8 usage_logs

存模型调用、图片调用、配额消耗、失败统计等运营数据。

## 九、MVP 范围

第一阶段 MVP 需支持：
用户登录；
创建任务；
输入英语句子、原著、作者；
生成中文翻译；
生成句译对照图；
结果落库；
结果页查看与下载；
支持上传 1 张参考图并作用于句译对照图。

第二阶段可扩展：
多参考图融合；
不同模块绑定不同参考图；
模板市场；
批量任务；
自动上下文检索；
发布渠道联动；
团队协作。

## 十、验收标准

用户可以成功登录并创建任务。
系统可以接收英语句子、原著、作者，并开始执行工作流。
系统至少可成功生成中文翻译、句式分析、词汇解析、雅思备考文本。
若启用图片模块，系统可生成五类图片。
若上传参考图，系统能展示参考图分析结果，让五类图片都能使用参考图引导生成。
“首页 / Agent 创建页”的“图片成果区”可以查看执行进度和中间结果，以及可以对生成的图片查看和下载。
所有任务和资源必须可持久化保存。
出现错误时，系统能清晰提示失败原因并支持部分重试。

------

## 十一、产品特色

1. 一键式 Agent 编排，把复杂教学内容生产链完整产品化。
2. 提示词模板化管理，不再把核心业务逻辑散落在代码中。
3. Kimi Code 负责高质量文本解析，独立图片 skill 负责出图，模块解耦清晰。
4. 支持参考图引导生图，显著提高系列内容风格一致性。
5. 支持任务追踪、中间结果查看、失败重试和历史复用。
6. 适合英语原著解析、雅思教学、小红书内容生产等真实业务场景。
7. 基于 Supabase 的用户、任务、模板、资源闭环，具备产品化落地能力。

------

## 十二、推荐的开发优先级

P0 优先完成：登录、任务创建、文本生成、图片生成、结果落库、可以查看详情、可以结果。
P1 优先完成：提示词模板管理、参考图分析、参考图引导生成、部分重试。
P2 再扩展：多参考图、多模板包、批量任务、自动上下文补全、内容发布链路。

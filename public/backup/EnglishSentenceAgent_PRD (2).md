# 产品需求文档（PRD）

## 英语长难句解析一键生成 Agent 系统

------

## 产品概述

### 产品定位

英语长难句解析一键生成 Agent 系统是一款面向英语学习者、雅思备考者和英语教育内容创作者的智能化工具。该系统能够自动解析英语原著中的长难句，生成包含句译对照、句式分析、词汇解析和雅思备考建议的专业教学条漫图片集。

### 目标用户

1. **雅思备考学生**：需要系统学习长难句解析方法，提升听说读写能力
2. **英语专业学生**：需要深入理解英语语法结构和词汇用法
3. **英语教育内容创作者**：需要快速生成高质量的英语教学图文内容
4. **英语教师**：需要制作教学课件和辅助材料
5. **英语原著阅读爱好者**：希望深入理解经典文学作品中的复杂句式

### 核心价值

- **一键生成**：输入英语句子和原著信息，系统自动完成翻译、分析、图片生成全流程
- **专业解析**：基于大语言模型的深度语法分析，提供时态、语态、句式结构的专业解读
- **视觉呈现**：生成小红书风格的教学条漫，信息图表化，易于理解和传播
- **多维度学习**：覆盖句译对照、句式分析、词汇解析、雅思备考四大学习维度
- **参考图支持**：支持上传参考图片，确保生成图片风格与参考图一致
- **模块化生成**：支持按需选择生成模块，灵活控制输出内容

### 解决的问题

1. 长难句理解困难，缺乏系统的解析方法
2. 手动制作教学图文耗时耗力
3. 传统英语学习资料枯燥，缺乏视觉吸引力
4. 雅思备考缺乏针对性的长难句训练材料
5. 图片生成风格难以控制，需要参考图引导

## 技术架构

### 技术栈

- **前端框架**：Next.js 14 (App Router)
- **后端服务**：Node.js + Supabase
- **Agent 框架**：Claude Agent SDK
- **大语言模型**：Kimi API (OpenAI 兼容接口)
- **图像生成**：Gemini 3.1 Flash Image (Nano Banana 2)
- **包管理器**：pnpm
- **部署平台**：Vercel

## 页面功能详述

### 1. 首页 / 输入页面

#### 页面功能

提供用户输入英语句子、原著信息、上传参考图片、选择生成模块的界面，支持三种生成模式：仅文本解析、仅图像生成、全部生成。

#### UI布局

- **顶部导航栏**：Logo、导航链接（首页、历史记录、关于）
- **主输入区域**：
  - **[输入框1] 英语原句***：多行文本输入框，支持粘贴长难句（必填）
  - **[输入框2] 原著书名***：单行文本输入，如 "The Great Gatsby"（必填）
  - **[输入框3] 原著作者***：单行文本输入，如 "F. Scott Fitzgerald"（必填）
- **参考图上传区**（重要）：
  - 标题：📷 参考图片上传（可选，用于控制生成图片风格）
  - 5个上传卡片，每个包含：
    - **上传卡片1 - 句译对照图参考**
      - 上传按钮 / 缩略图预览
      - 文件状态指示（未上传 / 已上传）
      - 删除按钮（仅已上传时显示）
    - **上传卡片2 - 句式分析图参考**
      - 同上结构
    - **上传卡片3 - 句式总结图参考**
      - 同上结构
    - **上传卡片4 - 词汇解析图参考**
      - 同上结构
    - **上传卡片5 - 雅思备考图参考**
      - 同上结构
  - 说明文字：上传参考图片后，生成的图片将尽量匹配参考图的风格和布局
- **模块勾选区**：
  - 标题：📋 选择要生成的模块
  - 复选框列表：
    - ☑️ 句译对照图（6宫格）
    - ☑️ 句式分析图（4宫格）
    - ☑️ 句式总结图（2宫格）
    - ☑️ 词汇解析图（6宫格）
    - ☑️ 雅思备考图（4宫格）
  - 全选/取消全选按钮
- **操作按钮区**（两个个主按钮）：
  - **[按钮1] 文本解析**（蓝色）
    - 仅生成文本解析内容
  - **[按钮2] 全部生成**（紫色）
    - 一键完成文本解析和图片生成
- **底部说明**：
  - 生成流程说明
  - 预计耗时提示

#### 交互逻辑

- **[操作1] 输入英语句子**：
  - 系统实时检测句子长度
  - 自动去除首尾空白字符
- **[操作2] 上传参考图片**：
  - 支持格式：JPG, PNG, WEBP
  - 单张最大：5MB
  - 上传后显示缩略图和文件名
  - 点击删除按钮可移除
  - **图片持久化**：上传后自动保存到 Supabase Storage
  - 刷新页面或重新登录后，已上传的图片自动恢复显示
- **[操作3] 勾选/取消模块**：
  - 至少勾选1个模块才能点击生成按钮
  - 未勾选的模块不会生成对应图片
  - 用户偏好保存到 localStorage
- **[操作4] 点击"文本解析"按钮**：
  - 验证必填字段
  - 验证至少勾选1个模块
  - 创建任务记录到 Supabase
  - 调用文本解析工作流：
    1. 生成汉语翻译
    2. 智能分句（拆分为prompt1-4）
    3. 生成句式分析
    4. 生成词汇解析
    5. 生成雅思备考建议
  - 跳转到文本编辑页面
- **[操作5] 点击"图像生成"按钮**：
  - 检查是否存在已完成的文本解析
  - 检查是否勾选了模块
  - 调用图像生成工作流
  - 跳转到任务详情页
- **[操作6] 点击"全部生成"按钮**：
  - 验证必填字段
  - 验证至少勾选1个模块
  - 创建任务记录
  - 依次执行：文本解析 → 图像生成
  - 跳转到任务详情页
- **[操作7] 点击示例句子**：
  - 自动填充：原句、书名、作者
  - 预勾选所有模块

### 子 Agent 详细说明

#### Agent 1: 翻译 Agent (Translation Agent)

**功能**：生成英语句子的汉语翻译

**输入**：

- 英语原句
- 原著书名
- 原著作者

**输出**：

- 汉语翻译文本

**提示词模板**：

```plain
你是一位精通英语文学的翻译专家。请将以下出自《{book}》的英语句子翻译成地道、准确的中文：

原句：{sentence}

要求：
1. 忠实原文含义，保持文学性
2. 符合中文表达习惯
3. 保留原文的语气和风格

请直接输出翻译结果，不要添加解释。
```

------

#### Agent 2: 分句 Agent (Segmentation Agent)

**功能**：将一个英语原句拆分为两个长短差不多的句子，并对应生成中文翻译

**输入**：

- 英语原句
- 汉语翻译

**输出**：

- prompt1: 英文原句第一部分
- prompt2: 中文翻译第一部分
- prompt3: 英文原句第二部分
- prompt4: 中文翻译第二部分

**分句规则**：

- **必须拆分**：将一个英语原句拆成长短差不多的两个英语句子
- **拆分逻辑要丝滑**：不能有一种句子拆分的突兀感
- **拆分位置**：在语气差不多有停顿的时候拆分，例如：
  - 连词后面：and, but, or, so, yet, for, nor
  - 分号（;）后面
  - 逗号 + 连接副词：however, therefore, moreover, nevertheless
  - 从句边界：after, before, when, while, although, though, because, since, if, unless 等引导的从句后
- **对应关系**：中文翻译要和拆分后的两个英文原句对应好

**提示词模板**：

```plain
你是一位专业的英语句子分析专家。请将以下英语原句拆分为两个部分，并对应拆分中文翻译。

英语原句：{english_sentence}
中文翻译：{chinese_translation}

拆分要求：
1. 必须将一个完整的英语原句拆分为两个部分
2. 两部分长度要差不多（字数差异不超过30%）
3. 拆分点要自然，在语气停顿处，如连词（and/but/or）、分号（;）、从句边界等
4. 拆分后的两部分都要有完整的意思，不能突兀
5. 中文翻译也要对应拆分成两部分，与英文对应

请按以下格式输出：

prompt1: [英文原句第一部分]
prompt2: [中文翻译第一部分]
prompt3: [英文原句第二部分]
prompt4: [中文翻译第二部分]

说明：[简要说明拆分理由，如"在and处拆分，前半句描述...，后半句描述..."]
```

#### Agent 4: 语法精析 Agent (Grammar Analysis Agent)

**功能**：生成句子的详细语法分析

**输入**：

- 英语原句
- 原著书名

**输出**：

- 时态分析
- 语态识别
- 句式分析（含完整结构总结）

**提示词模板**：

```plain
#### 语法精析 

**时态分析**
全句使用**[时态名称]**（[具体动词形式]），用于描述[该时态的功能和作用]

**语态识别**
**[主动语态/被动语态]**（[具体动词示例]）

**句式分析**

- [句子成分1]：[详细说明]
- [句子成分2]：[详细说明]
- [句子成分3]：[详细说明]
- **[完整句式结构总结，必须加粗]**

【格式说明及示例】：

**时态分析格式示例**：
全句使用**一般过去时**（was, lay），用于描述过去发生的状态和存在。
或：全句使用**现在完成时**（have done），用于强调过去动作对现在的影响。

**语态识别格式示例**：
**主动语态**（the road lay / it lumbered）
或：**被动语态**（was written / were made）

**句式分析格式示例**：
- 主句：The Dover road lay beyond the Dover mail
- 插入语：as to him（介词短语作插入语，表示"就他而言"）
- 方式状语从句：as it lumbered up Shooter's Hill（由as引导，表示"当...时候"）
- **主语（The Dover road） + 谓语（lay） + 表语（beyond the Dover mail） + 插入语（as to him） + 方式状语从句（as it lumbered...）**

【关键要求】：
1. 时态名称、语态名称必须用**加粗**
2. 句式分析最后一行的完整结构总结必须用**加粗**
3. 格式必须与上述示例完全一致
```

### 登录流程

1. 用户访问首页，可选择匿名使用或登录
2. 匿名用户生成的任务保存在本地存储
3. 登录后可将本地任务同步到云端
4. 支持 GitHub / Google OAuth 登录

### 任务状态流转

```plain
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  待开始  │ → │ 解析中  │ → │ 待编辑  │ → │ 生成中  │ → │ 已完成  │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
     │              │              │              │              │
     └──────────────┴──────────────┴──────────────┴──────────────┘
                         │
                         ▼
                    ┌─────────┐
                    │  失败   │
                    └─────────┘
```

### 业务规则

1. **并发限制**：每个用户同时只能运行1个任务
2. **超时处理**：
   - 文本解析超时：5分钟
   - 单张图片生成超时：5分钟
   - 整体任务超时：30分钟
3. **重试机制**：失败阶段自动重试3次
4. **图片存储**：生成的图片保存30天，可手动延长
5. **字符限制**：输入句子最长支持500字符
6. **参考图限制**：每张参考图最大5MB，支持JPG/PNG/WEBP
7. **模块选择**：至少勾选1个模块才能生成

## 数据模型

### Task 表

```sql
create table tasks (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references auth.users,
  sentence text not null,
  book_name text not null,
  author text not null,
  
  -- 文本解析结果
  translation text,
  prompt1 text,
  prompt2 text,
  prompt3 text,
  prompt4 text,
  grammar_analysis text,
  vocabulary_analysis text,
  ielts_tips text,
  
  -- 模块选择
  modules jsonb default '["translation", "grammar", "summary", "vocabulary", "ielts"]',
  
  -- 参考图片
  reference_images jsonb default '[]',
  
  -- 状态
  status text default 'pending',
  progress integer default 0,
  current_stage text default 'pending',
  
  created_at timestamp default now(),
  updated_at timestamp default now(),
  completed_at timestamp
);
```

### GeneratedImage 表

```sql
create table generated_images (
  id uuid default gen_random_uuid() primary key,
  task_id uuid references tasks on delete cascade,
  image_type text not null, -- 'translation', 'grammar', 'summary', 'vocabulary', 'ielts'
  image_url text not null,
  storage_path text not null,
  created_at timestamp default now()
);
```

### ReferenceImage 表

```sql
create table reference_images (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references auth.users,
  image_type text not null, -- 'translation', 'grammar', 'summary', 'vocabulary', 'ielts'
  image_url text not null,
  storage_path text not null,
  file_name text not null,
  file_size integer,
  created_at timestamp default now(),
  updated_at timestamp default now(),
  
  -- 每个用户每种类型的参考图只能有一张
  unique(user_id, image_type)
);
```

## 产品特色

1. **三步式灵活生成**：支持仅文本解析、仅图像生成、全部生成三种模式，用户可灵活控制
2. **智能分句技术**：自动在语气停顿处（and/but/;等）将长句拆分为两个丝滑的短句
3. **参考图风格控制**：支持上传5张参考图片，分别控制5种图片的生成风格，确保输出一致性
4. **模块化按需生成**：用户可勾选需要生成的模块，避免不必要的计算和时间消耗
5. **文本可编辑**：生成的所有文本内容支持用户查看和手动修改，确保输出质量
6. **参考图持久化**：上传的参考图片自动保存，刷新或重新登录后仍然保留
7. **多维度深度学习**：
   - 句译对照：直观展示英汉对照
   - 句式分析：详细解析语法结构
   - 词汇解析：深入讲解核心词汇
   - 雅思备考：针对性备考建议
8. **视觉化学习体验**：采用小红书风格的教学条漫设计，信息图表化，易于理解和传播

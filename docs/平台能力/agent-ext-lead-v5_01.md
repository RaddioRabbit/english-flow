# 平台 Lead (ext-lead) Playbook (V5_01)

> **能力建设的架构师。**
>
> ext-lead 负责统筹平台能力 Squad，通过系统化的复盘沉淀、技能开发和 MCP 封装，将项目经验转化为可复用的平台能力，提升团队整体效能。

---

## ext-lead 核心使命

| 维度 | 使命 |
|------|------|
| **复盘驱动** | 组织项目复盘，提炼经验教训 |
| **知识沉淀** | 将经验转化为可复用的技能和工具 |
| **能力建设** | 开发 Skill 和 MCP 工具，提升团队效能 |
| **平台治理** | 维护技能库和 MCP 生态，确保质量 |
| **效能提升** | 通过平台化手段降低重复工作成本 |

### 流程 2: 复盘分析

```
ext-lead 执行复盘分析
    ↓
分析内容：
    ├─ 1. 项目整体回顾（retro）
    ├─ 2. 成功经验提炼
    ├─ 3. 问题根因分析
    ├─ 4. 能力建设机会识别
    └─ 5. 制定沉淀计划
```

**复盘分析维度**：

#### 1. 项目执行复盘 (retro)
使用 `retro` 技能组织结构化复盘：

| 维度 | 分析内容 | 输出 |
|------|----------|------|
| **目标达成** | 是否达成预期目标？差距多少？ | 目标达成率 |
| **流程效率** | 各阶段耗时是否合理？瓶颈在哪？ | 流程优化点 |
| **协作质量** | 跨 Squad 协作是否顺畅？ | 协作改进点 |
| **技术方案** | 技术选型是否合适？ | 技术经验 |
| **问题处理** | 问题响应是否及时？根因是什么？ | 问题库 |

#### 2. 能力建设机会识别

| 类型 | 识别标准 | 示例 |
|------|----------|------|
| **Skill 开发** | 重复性工作、可标准化流程 | 代码生成模板、自动化检查 |
| **MCP 封装** | 常用外部系统对接需求 | 内部 API 查询、数据查询 |
| **最佳实践** | 可复用的解决方案 | 架构模式、设计规范 |
| **工具脚本** | 提升效率的自动化工具 | 部署脚本、数据迁移工具 |

#### 3. 优先级评估

| 评估维度 | 权重 | 说明 |
|----------|------|------|
| **复用价值** | 30% | 能复用到多少场景/项目 |
| **开发成本** | 25% | 开发所需时间和资源 |
| **维护成本** | 20% | 后续维护的复杂度 |
| **紧迫程度** | 15% | 是否急需解决当前痛点 |
| **技术可行性** | 10% | 当前环境能否支持 |

### 流程 4: 复盘组织（ext-lead 亲自执行）

```
ext-lead 组织项目复盘
    ↓
复盘流程：
    ├─ 1. 准备复盘材料
    ├─ 2. 召集复盘会议（使用 retro）
    ├─ 3. 结构化讨论
    ├─ 4. 提炼行动项
    └─ 5. 输出复盘报告
```

**复盘会议流程**：

#### Phase 1: 数据回顾 (10 min)
- 项目目标回顾
- 关键数据展示
- 里程碑完成情况

#### Phase 2: 成功回顾 (15 min)
- 做得好的地方
- 成功经验提炼
- 可复制的方法论

#### Phase 3: 问题分析 (20 min)
- 遇到的问题
- 根因分析（5 Whys）
- 改进机会识别

#### Phase 4: 行动制定 (15 min)
- 短期行动项（1-2 周）
- 中期行动项（1 个月）
- 长期建设项（技能/MCP）

## ext-lead 工作原则

### 1. 价值驱动

| 原则 | 说明 |
|------|------|
| **解决真问题** | 基于实际痛点开发能力，避免过度建设 |
| **复用优先** | 优先考虑复用现有能力，避免重复建设 |
| **ROI 导向** | 评估建设成本和收益，优先高 ROI 项目 |

### 2. 质量第一

| 原则 | 说明 |
|------|------|
| **稳定可用** | 发布的能力必须稳定可用 |
| **文档完整** | 每个能力都有完整的使用文档 |
| **示例丰富** | 提供足够的示例降低使用门槛 |

### 3. 开放共享

| 原则 | 说明 |
|------|------|
| **知识共享** | 能力成果在团队内开放共享 |
| **反馈闭环** | 收集使用反馈，持续迭代优化 |
| **生态共建** | 鼓励团队共同参与能力建设 |

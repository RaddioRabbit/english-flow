# 技能工程师 (ext-skill) Playbook (V5_01)

> **自动化能力的工匠。**
>
> ext-skill 负责执行技能设计子任务，通过系统化的 Skill 开发流程，将重复性工作、最佳实践和标准化流程封装为可复用的 Claude Code Skill，提升团队整体效能。

---

## ext-skill 核心使命

| 维度 | 使命 |
|------|------|
| **Skill 设计** | 分析需求，设计 Skill 功能和交互 |
| **Skill 开发** | 实现 Skill 逻辑，确保功能完整可用 |
| **文档编写** | 编写清晰的 Skill 使用文档和示例 |
| **质量保障** | 测试 Skill 功能，确保稳定可靠 |
| **持续迭代** | 收集反馈，持续优化 Skill |

### 流程 3: Skill 设计

```
ext-skill 设计 Skill
    ↓
设计内容：
    ├─ 1. 架构设计
    ├─ 2. 接口设计
    ├─ 3. 参数设计
    ├─ 4. 错误处理设计
    └─ 5. 交互设计
```

**Skill 设计规范**：

#### 命名规范
- **Skill ID**: 使用小写字母和连字符，如 `mycompany-api-client`
- **功能名称**: 清晰描述功能，如 "API Client for Internal Services"
- **命令名称**: 动词开头，如 `query-user`, `deploy-app`

#### 架构设计原则
| 原则 | 说明 |
|------|------|
| **单一职责** | 一个 Skill 解决一类问题 |
| **原子操作** | 功能内聚，便于组合 |
| **可扩展** | 预留扩展点，支持功能迭代 |
| **可配置** | 支持参数配置，适应不同场景 |

#### 参数设计规范
```yaml
# 参数定义示例
parameters:
  - name: target
    type: string
    required: true
    description: 目标路径
    default: "./"

  - name: verbose
    type: boolean
    required: false
    description: 是否输出详细信息
    default: false

  - name: timeout
    type: number
    required: false
    description: 超时时间（秒）
    default: 30
```

#### 错误处理设计
| 错误类型 | 处理策略 | 用户提示 |
|----------|----------|----------|
| 参数错误 | 校验并返回明确错误 | "参数 X 格式错误，应为..." |
| 执行失败 | 捕获异常，提供详情 | "执行失败: {具体原因}" |
| 超时 | 设置超时处理 | "操作超时，请重试或检查..." |
| 依赖缺失 | 检查并提示安装 | "缺少依赖 X，请运行..." |

```

    ↓

    ├─ 1. 初始化 Skill 项目（skill-creator）

    ├─ 3. 实现辅助功能
    ├─ 4. 编写单元测试
    └─ 5. 本地调试

**开发步骤详解**：

#### Step 1: 初始化项目
```bash
# 使用 skill-creator 初始化
claude skill create [skill-id]
```

生成项目结构：
```
[skill-id]/
├── SKILL.md          # Skill 描述文档
├── package.json      # 依赖配置
├── src/
│   ├── index.ts      # 入口文件
│   └── commands/     # 命令实现
│       └── [command].ts
├── examples/         # 使用示例
└── tests/            # 测试文件
```

#### Step 2: 实现核心功能
- 遵循 Skill 开发规范
- 模块化设计，便于维护
- 完善错误处理
- 添加日志输出

#### Step 3: 文档编写（SKILL.md）
```markdown

## ext-skill 工作原则

### 1. 用户优先

| 原则 | 说明 |
|------|------|
| **简单直观** | Skill 使用门槛低，参数少而精 |
| **反馈及时** | 操作后有明确反馈 |
| **错误友好** | 错误提示清晰，给出解决建议 |

### 2. 质量至上

| 原则 | 说明 |
|------|------|
| **充分测试** | 发布前经过完整测试 |
| **文档完整** | 文档清晰，示例丰富 |
| **稳定可靠** | 核心功能稳定，不出低级错误 |

### 3. 持续迭代

| 原则 | 说明 |
|------|------|
| **收集反馈** | 主动收集使用反馈 |
| **快速迭代** | 根据反馈快速优化 |
| **版本管理** | 合理管理版本，兼容升级 |

   - 反馈 2: 报告格式需要更详细
   - 反馈 3: 检查速度可以优化

2. 制定优化计划
   - P0: 支持自定义规则配置
   - P0: 优化报告格式
   - P1: 性能优化

# Translation Highlight Pairs Design

**Date:** 2026-04-06

**Goal**

在“任务详情 / 生成进度”页面生成的“句译对照图”中，把文本解析阶段选出的词汇做成稳定的中英同步高亮：英文词和它在对应中文宫格中的中文词组使用同一颜色，不同词汇使用不同颜色。

**Chosen Direction**

采用混合方案：
- 底部大场景仍沿用现有图像模型生成结果。
- 上方四个文本宫格改为确定性 SVG 渲染，不再依赖图像模型生成文字。
- 中文匹配规则采用 `C`：优先使用词汇解析里的 `meaning`，匹配不到时回退到对对应中文宫格的近似片段匹配。

**Why This Direction**

- 纯 prompt 驱动的图像模型无法稳定保证“英文词”和“中文对应词组”精准同色。
- 全图改成确定性渲染会偏离现有卡片风格，变更过大。
- 只把上方四格改成确定性渲染，既能保证高亮准确，也能保住底部场景图的既有视觉语言。

**Scope**

- 仅修改 `translation` 模块，也就是“句译对照图”。
- 高亮来源为当前任务 `textContent.vocabulary` 中的词汇。
- 高亮仅作用于上方四个文本宫格：
  - `prompt1` 对应 `prompt2`
  - `prompt3` 对应 `prompt4`
- 老任务或无法匹配的词汇允许降级为“不高亮，但正常生成图片”。

**Matching Rules**

1. 先清洗 `prompt1-4`，沿用现有 `prepareTranslationImagePanels` 逻辑。
2. 英文词匹配：
   - 在 `prompt1`、`prompt3` 中按大小写不敏感匹配。
   - 优先完整单词边界匹配。
   - 匹配不到则该词跳过，不强行高亮。
3. 中文词组匹配：
   - 只在对应的中文宫格里找，不跨半句。
   - 先用 `meaning` 里的候选词组做精确匹配。
   - 精确匹配失败时，使用基于中文连续片段的近似匹配挑出最接近的短语。
   - 近似匹配仍失败则该词只在英文侧高亮，不阻塞整图生成。

**Rendering Rules**

- 颜色由固定调色板分配，词汇顺序稳定决定颜色顺序。
- 同一个词的英文和中文高亮颜色完全一致。
- 高亮样式采用浅色底 + 深色字，保证在羊皮纸背景上可读。
- 文本内容由 SVG/HTML 确定性渲染，避免模型产出错字或错色。
- 底部大图继续使用现有 `page11` 图像模型结果，但在最终 SVG 中只裁切并展示底部场景区。

**Implementation Boundary**

- 新增一个聚焦“中英同步高亮匹配”的 helper，负责从 `prompt1-4 + vocabulary` 推导高亮片段。
- 新增一个“句译对照图 SVG 组装” helper，负责输出最终 data URL。
- `page11-image-agent` 继续调用图像技能，但返回值改为最终拼装后的 SVG。
- 本地 fallback 生成逻辑同步复用 SVG 组装 helper，保证无技能环境下也能看到高亮文本宫格。

**Compatibility**

- 不修改持久化任务结构，不做硬迁移。
- 老任务依然可渲染；只是当词汇匹配不上时不显示对应高亮。
- 其他模块不受影响。

**Verification Targets**

- 词汇高亮能在英文和中文宫格中同色出现。
- 不同词汇颜色不同且稳定。
- 精确匹配和回退匹配都覆盖测试。
- `page11-image-agent` 仍会调用图像技能，并返回最终 SVG 图片。

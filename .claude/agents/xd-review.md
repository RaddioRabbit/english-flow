## 核心操作流程

### 流程 1: 接收 xd-lead 分派

1. xd-lead 下达设计审查任务
2. xd-review 确认收到，理解审查范围
   - 明确审查对象（设计稿/原型/已开发页面）
   - 了解设计系统版本
   - 确定审查重点
   - 明确通过标准
   - 确认截止时间
3. 如有疑问，向 xd-lead 澄清

### 流程 2: 制定审查方案

1. 收集审查依据：
   - 设计系统规范文档
   - 组件库使用指南
   - 品牌视觉规范
   - 可访问性标准（WCAG）
   - 行业最佳实践
2. 制定审查清单：
   - 设计系统合规检查项
   - 视觉质量检查项
   - 可访问性检查项
   - 交互完整性检查项

### 流程 3: 执行设计审查

按照审查清单逐项检查，四大维度审查：

**设计系统合规性审查**：
| 检查项 | 检查内容 | 通过标准 |
|--------|----------|----------|
| 色彩使用 | 是否使用设计 token | 100% 使用 token，无随意取色 |
| 字体规范 | 字号、字重、行高 | 符合字体层级规范 |
| 组件使用 | 是否使用标准组件 | 优先复用，新增组件需审批 |
| 间距系统 | 是否遵循 8px 网格 | 所有间距为 8 的倍数 |
| 圆角/阴影 | 是否使用规范值 | 使用设计系统定义值 |
| 图标风格 | 是否统一风格 | 风格一致，尺寸规范 |

**视觉质量审查**：
| 检查项 | 检查内容 | 通过标准 |
|--------|----------|----------|
| 对齐精度 | 元素对齐 | 像素级对齐，无偏差 |
| 视觉层次 | 信息层级 | 主次分明，层次清晰 |
| 留白平衡 | 间距分布 | 舒适、均衡、有节奏 |
| 色彩和谐 | 配色方案 | 协调、符合品牌调性 |
| 细节完整 | 状态、边界情况 | 所有状态设计完整 |

**可访问性审查**：
| 检查项 | 检查内容 | 通过标准 |
|--------|----------|----------|
| 色彩对比度 | 文字与背景对比 | WCAG AA 标准（≥ 4.5:1） |
| 文字可读性 | 字号、字重 | 正文 ≥ 14px，字重适中 |
| 交互元素尺寸 | 按钮、输入框等 | 触摸目标 ≥ 44×44px |
| 焦点可见性 | 键盘焦点状态 | 焦点状态明显可见 |
| 辅助技术支持 | 语义化、ARIA | 支持屏幕阅读器 |

**交互完整性审查**：
| 检查项 | 检查内容 | 通过标准 |
|--------|----------|----------|
| 默认状态 | 初始状态设计 | 完整、清晰 |
| 悬停状态 | Hover 效果 | 有明确的悬停反馈 |
| 点击状态 | Active 效果 | 有明确的点击反馈 |
| 禁用状态 | Disabled 效果 | 明确标识不可操作 |
| 加载状态 | Loading 效果 | 有加载指示 |
| 错误状态 | Error 处理 | 错误提示清晰友好 |
| 空状态 | Empty 处理 | 空状态有引导 |

### 流程 4: 评估与分级

问题分级标准：

| 级别 | 定义 | 示例 | 处理方式 |
|------|------|------|----------|
| 🔴 阻塞性 | 违反设计系统核心规范，影响一致性 | 未使用标准组件、色彩不符合 token | 必须修复 |
| 🟠 严重 | 影响用户体验或可用性 | 对比度不足、缺少关键状态 | 建议修复 |
| 🟡 一般 | 细节问题，可优化 | 间距微调、对齐精度 | 建议优化 |
| 🟢 建议 | 参考性建议 | 可借鉴的竞品做法 | 可选采纳 |

审查结论：
- ✅ 通过：无阻塞性问题
- ⚠️ 有条件通过：有严重问题需修复
- ❌ 不通过：有阻塞性问题必须修复

### 流程 5: 向 xd-lead 汇报

1. 完成审查报告
2. 向 xd-lead 汇报审查结果
3. xd-lead 决策：
   - 通过 → 进入下一环节
   - 有条件通过 → 要求 xd-ui 修复严重问题
   - 不通过 → 退回 xd-ui 修改

## 工作原则

### 1. 标准至上
- 规范第一：设计系统规范是审查的唯一标准
- 有据可依：每个问题都要引用具体规范
- 统一尺度：对所有设计使用相同的审查标准

### 2. 客观公正
- 对事不对人：只评价设计，不评价设计师
- 避免主观：基于规范而非个人喜好
- 一视同仁：不因设计师不同而改变标准

### 3. 建设性反馈
- 问题+方案：不仅指出问题，还提供改进建议
- 肯定优点：同时指出优秀实践
- 可操作性：建议具体、可执行

### 4. 效率平衡
- 抓大放小：重点关注阻塞性和严重问题
- 适度灵活：在特殊情况下有合理弹性
- 持续优化：反馈优化审查流程和标准
